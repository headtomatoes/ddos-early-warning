The dataset presents a severe multi-class imbalance. The primary modeling challenge is not detecting attacks (which dominate), but correctly identifying the 113,828 BENIGN flows without triggering false positives. SMOTE is impractical at 70M rows; class-weighted loss (scale_pos_weight in XGBoost) is the primary strategy. SMOTE will be applied only as an ablation comparison on a down-sampled subset.

03Preprocessing & Feature Engineering Strategy
D1
Drop identifiers & leakage vectors
Remove Flow ID, Source IP, Destination IP, Source Port, Timestamp, Unnamed: 0. These are not flow-level behavioral features.
D2
Drop exact duplicates (r = 1.0000)
9 feature pairs are mathematically identical. Keep one from each: drop Subflow Fwd/Bwd Packets, Subflow Fwd/Bwd Bytes, Avg Fwd/Bwd Segment Size, Fwd Packets/s, RST Flag Count, Fwd Header Length.1.
D3
Drop zero-variance / no-separation features
Features whose histogram shows all 19 classes collapsing into a single spike at zero with no inter-class spread:
Bwd IAT Total Fwd IAT Total Fwd Avg Bytes/Bulk Fwd Avg Packets/Bulk Fwd Avg Bulk Rate Bwd Avg Bytes/Bulk Bwd Avg Packets/Bulk Bwd Avg Bulk Rate SimillarHTTP Inbound

Note: Fwd IAT Total is dropped because r=0.9999 with Flow Duration; Flow Duration is retained as the primary temporal feature.

M1
Impute missing values
Flow Bytes/s and Flow Packets/s have 3.08% nulls originating from infinity replacement (division-by-zero in zero-duration flows). Strategy: impute with per-class median to avoid cross-class contamination. Set aside NaN rows for post-imputation audit.
T1
Log-transform right-skewed features
All retained IAT, byte-count, and rate features show severe right skew (massive spike at zero, long tail). Apply log1p(x) — safe for zero values. Features requiring transformation:
Flow Bytes/s · Flow Packets/s
Flow IAT Mean · Flow IAT Std
Fwd/Bwd IAT Mean/Std/Max/Min
Total Length Fwd/Bwd Packets
Total Fwd/Bwd Packets
Flow Duration · Packet Length Variance
S1
Scale after transform
Apply MinMaxScaler fitted only on training split to prevent data leakage. Final pipeline: log1p → MinMaxScaler → model. Persist scaler.pkl for ONNX pipeline.
04Noteworthy Insights & Next Steps
The real minority class problem is BENIGN, not attack types
All 18 attack classes together represent 99.8% of data. The model must learn what is not an attack from only 113,828 examples. Standard accuracy metrics will be misleading — a model predicting "attack" for every row achieves 99.8% accuracy. F1-score on the BENIGN class specifically must be the primary evaluation metric alongside macro-F1. FPR target (<0.1%) must be validated exclusively against BENIGN flows.
Protocol feature is negatively correlated with ACK Flag Count (r = −0.993)
UDP-based attacks (TFTP, DrDoS_UDP, DrDoS_DNS, DrDoS_SNMP) have Protocol=17 and generate zero ACK flags by definition. This creates a near-deterministic rule: Protocol=17 + ACK=0 is almost certainly an attack. A tree-based model will discover this instantly, but it also means the model may be implicitly learning protocol-level rules rather than behavioral patterns. Run an ablation with Protocol excluded to measure the behavioral-only F1 impact.
Total Backward Packets is a discrete 0–6 integer — treat as categorical
The histogram shows only values 0–6 across all 70M rows with no continuous distribution. This is not a continuous feature and should not be log-transformed. Encode as an ordinal integer directly. Syn floods are almost exclusively 0 (no server response completes), while reflection attacks show values 1–4. This single feature, combined with Flow Bytes/s, may be sufficient to distinguish Syn floods from all DrDoS variants with high precision.
